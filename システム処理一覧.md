この部分は操作が多いので、顧客向け仕様では **4つの機能ブロックに分けると整理しやすい**です。

1. 編集モーダル表示
2. モーダル操作（キャンセル・適用・削除）
3. コンテキストメニュー（右クリック）
4. 削除確認

それぞれを提示された **処理名 / トリガー分類 / トリガー条件 / 処理内容** で書くと次のようになります。

---

## 編集モーダル表示

| 処理名            | トリガー分類（イベント） | トリガー条件                      | 処理内容                                    |
| -------------- | ------------ | --------------------------- | --------------------------------------- |
| イベント編集モーダル表示   | ダブルクリック      | カレンダー上のイベントをダブルクリックしたとき     | イベント編集モーダルを表示し、イベント情報を表示する。削除ボタンを有効にする。 |
| 新規イベント入力モーダル表示 | ダブルクリック      | カレンダーのイベントがない場所をダブルクリックしたとき | イベント入力モーダルを表示する。削除ボタンは無効状態で表示する。        |

---

## モーダル操作

| 処理名          | トリガー分類 | トリガー条件             | 処理内容                   |
| ------------ | ------ | ------------------ | ---------------------- |
| 編集キャンセル      | ボタン押下  | モーダルで「キャンセル」を押したとき | モーダルを閉じ、編集内容は反映しない。    |
| 編集結果反映       | ボタン押下  | モーダルで「適用」を押したとき    | 入力内容をカレンダーに反映する（仮反映）。  |
| イベント削除（モーダル） | ボタン押下  | モーダルで「削除」を押したとき    | イベントをカレンダーから削除する（仮反映）。 |

※ここでは **保存ボタンを押すまではサーバー保存されない**という状態を「仮反映」と表現しています。

---

## コンテキストメニュー（右クリック）

| 処理名          | トリガー分類 | トリガー条件                    | 処理内容               |
| ------------ | ------ | ------------------------- | ------------------ |
| 新規作成メニュー表示   | 右クリック  | カレンダーのイベントがない場所を右クリックしたとき | 「新規作成」メニューを表示する。   |
| イベント操作メニュー表示 | 右クリック  | カレンダー上のイベントを右クリックしたとき     | 「編集」「削除」メニューを表示する。 |

---

## メニュー操作

| 処理名        | トリガー分類  | トリガー条件             | 処理内容                        |
| ---------- | ------- | ------------------ | --------------------------- |
| 新規イベント作成   | メニュー選択  | 「新規作成」を選択したとき      | イベント入力モーダルを表示する。            |
| イベント編集     | メニュー選択  | 「編集」を選択したとき        | イベント編集モーダルを表示する。            |
| イベント削除確認表示 | メニュー選択  | 「削除」を選択したとき        | 削除確認ダイアログを表示する。             |
| イベント削除実行   | ダイアログ操作 | 削除確認ダイアログでOKを押したとき | イベントを削除し、カレンダー表示に反映する（仮反映）。 |

---

## 状態補足（仕様として書くと良い）

処理一覧とは別に、仕様として次を1行書くと分かりやすくなります。

仮反映仕様

* 編集・追加・削除の結果はカレンダー画面に一時的に反映される。
* 「保存」ボタンを押すまではサーバーには保存されない。

---

この画面は機能が多いですが、整理すると実際は **4種類の操作**しかありません。

* ダブルクリック操作
* 右クリック操作
* モーダル操作
* 削除確認操作

このように **操作タイプごとにまとめると、処理一覧が非常に整理しやすくなります。**
